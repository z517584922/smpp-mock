# SMPP Mock 服务器部署指南

## 服务器信息
- IP地址：123.253.110.98
- 操作系统：Ubuntu 22.04

## 1. 环境准备

### 1.1 安装 Node.js 和 pnpm
```bash
# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 pnpm
curl -fsSL https://get.pnpm.io/install.sh | sh -
```

### 1.2 安装项目依赖
```bash
# 克隆项目
git clone <项目仓库地址>
cd smpp-mock

# 安装依赖
pnpm install
```

## 2. 环境配置

### 2.1 创建 .env 文件
在项目根目录创建 `.env` 文件，配置以下环境变量：
```env
# SMPP 服务器配置
SMPP_HOST=0.0.0.0
SMPP_PORT=2775
SMPP_SYSTEM_ID=your_system_id
SMPP_PASSWORD=your_password

# 日志配置
LOG_LEVEL=info
LOG_FILE_PATH=logs/smpp-mock.log
LOG_FILE_MAX_SIZE=10485760
LOG_FILE_MAX_FILES=5
```

## 3. 构建和运行

### 3.1 构建项目
```bash
pnpm run build
```

### 3.2 运行服务器
```bash
pnpm start
```

## 4. 进程管理（可选）

### 4.1 使用 PM2 管理进程
```bash
# 安装 PM2
npm install -g pm2

# 启动服务
pm2 start dist/index.js --name smpp-mock

# 查看状态
pm2 status

# 设置开机自启
pm2 startup
pm2 save
```

## 5. 防火墙配置

确保服务器防火墙允许 SMPP 端口（默认 2775）的访问：
```bash
sudo ufw allow 2775/tcp
```

## 6. 验证部署

1. 检查服务是否正常运行：
```bash
netstat -tulpn | grep 2775
```

2. 查看日志文件：
```bash
tail -f logs/smpp-mock.log
```

## 7. 注意事项

1. 确保 `.env` 文件中的配置信息正确，特别是 `SMPP_SYSTEM_ID` 和 `SMPP_PASSWORD`
2. 日志文件默认保存在 `logs` 目录下，请确保该目录存在且有写入权限
3. 建议使用 PM2 等进程管理工具来确保服务的持续运行
4. 定期检查日志文件大小，避免磁盘空间耗尽

## 8. 故障排除

如果遇到问题，请检查：
1. 日志文件中的错误信息
2. 端口是否被占用
3. 防火墙设置是否正确
4. 环境变量配置是否正确
